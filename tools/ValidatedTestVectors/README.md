# ValidatedTestVectors

A standalone Lean 4 / Mathlib project that formally proves the round-trip
correctness of the APRS Base91 compressed-position encoding used by the rest of
this repository (an Arduino/PlatformIO C++ library), and emits the proven
encoded bytes as a JSON oracle for the C++ test suite.

## What is proved

`ValidatedTestVectors/Basic.lean` defines `encodeLat` / `decodeLat` and
`encodeLon` / `decodeLon` over `ℚ`, and proves:

- `encodeLat_decodeLat_roundtrip` — for any `x ∈ [-90, 90]`,
  `|decodeLat (encodeLat x) - x| ≤ 1/300000`.
- `encodeLon_decodeLon_roundtrip` — for any `x ∈ [-180, 180]`,
  `|decodeLon (encodeLon x) - x| ≤ 1/150000`.

The proofs certify the *mathematical* formula. The C++ implementation uses
floats; that rounding error is a separate concern not covered here.

## What is generated

`Main.lean` evaluates `encodeLat` / `encodeLon` at the fixed-point sites in its
`fixedPoints` list (Munich, Cape Town, Auckland, North Pole, Equator-0) and
writes `proved_vectors.json`. That JSON is consumed by
`tools/gen_aprs_vectors.py`, whose `FIXED_POINTS` list must stay in lock-step.

## Build

From this directory:

```sh
lake build ValidatedTestVectors    # type-check the library — this *is* the proof suite
lake env lean --run Main.lean      # regenerate proved_vectors.json
```

Type-checking is the test suite: if `lake build ValidatedTestVectors` exits
clean, the proofs are valid. There is no separate `lake test` step.

The first build downloads Mathlib and may take 10+ minutes (a few minutes if
the Mathlib build cache is reachable via `lake exe cache get`). Subsequent
builds are incremental.

`lake build` (no target) additionally tries to link the `validated_test_vectors`
executable, which fails on macOS with a clang `execve` error — see
`AGENTS.md` for the explanation and workarounds. Use `lake env lean --run
Main.lean` to regenerate the JSON on macOS.

## Project layout

```
.
├── lakefile.toml                       # Lake manifest; project-wide [leanOptions]
├── lean-toolchain                      # leanprover/lean4:v4.28.0 (must match Mathlib rev)
├── lake-manifest.json                  # committed; regenerated by `lake update`
├── Main.lean                           # exe entry + JSON generator
├── ValidatedTestVectors.lean           # library root (re-exports Basic)
├── ValidatedTestVectors/
│   └── Basic.lean                      # definitions + round-trip proofs
├── proved_vectors.json                 # generated, committed
└── AGENTS.md                           # detailed guidance for coding assistants
```

For deeper context — proof structure, the macOS link gotcha, known limitations,
and how to extend the proofs — see `AGENTS.md`.
