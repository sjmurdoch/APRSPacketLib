# ValidatedTestVectors

A standalone Lean 4 / Mathlib project that formally proves the round-trip
correctness of the APRS Base91 compressed-position encoding used by the rest of
this repository (an Arduino/PlatformIO C++ library), and acts as the runtime
validator (encode + decode) for the JSON test vectors produced by
`tools/gen_aprs_vectors.py`.

## What is proved

`ValidatedTestVectors/Basic.lean` defines `encodeLat` / `decodeLat` and
`encodeLon` / `decodeLon` over `‚Ñö`, and proves:

- `encodeLat_decodeLat_roundtrip` ‚Äî for any `x ‚àà [-90, 90]`,
  `|decodeLat (encodeLat x) - x| ‚â§ 1/300000`.
- `encodeLon_decodeLon_roundtrip` ‚Äî for any `x ‚àà [-180, 180]`,
  `|decodeLon (encodeLon x) - x| ‚â§ 1/150000`.

The proofs certify the *mathematical* formula. The C++ implementation uses
floats; that rounding error is a separate concern not covered here.

## What is validated

`Main.lean` reads pipe-delimited rows
(`name|lat_deg|lon_deg|base91_lat|base91_lon`) from stdin, runs
`encodeLat`/`encodeLon` (byte-exact) and `decodeLat`/`decodeLon` (within the
proven `1/factor` bound) per row, and prints `OK <name>` or
`FAIL <name>: <reason>`. Exits 1 if any row fails.

The fixed-point list lives in `tools/aprs_vectors.json`; the
`tools/gen_aprs_vectors.py validate-lean` subcommand pipes the rows into Lean
over stdin.

## Build

From this directory:

```sh
lake build ValidatedTestVectors                                  # type-check the library ‚Äî this *is* the proof suite
echo 'Munich|48.138630|11.573410|6/Y`|QG2.' | \
    lake env lean --run Main.lean                                # run the validator
```

Type-checking is the test suite: if `lake build ValidatedTestVectors` exits
clean, the proofs are valid. There is no separate `lake test` step.

The first build downloads Mathlib and may take 10+ minutes (a few minutes if
the Mathlib build cache is reachable via `lake exe cache get`). Subsequent
builds are incremental.

`lake build` (no target) additionally tries to link the `validated_test_vectors`
executable, which fails on macOS with a clang `execve` error ‚Äî see
`AGENTS.md` for the explanation and workarounds. Use `lake env lean --run
Main.lean` to run the validator on macOS.
